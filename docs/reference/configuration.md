# Configuration

Optional per-directory pipeline control via `doc-convert.yml` in the **current working directory** when you run `doc-convert` (conversion or `--list-steps`). The file is **not** read from `--project-root`.

## Create a config file

From the directory where you run conversions:

```bash
cd ../docx-examples/pdc
bun ../../doc-convert/src/index.ts configure
```

Or, with a linked `doc-convert` binary on your `PATH`:

```bash
cd path/to/your/project
doc-convert configure
```

This writes `./doc-convert.yml` with every **configurable** step set to `run` (12 steps). Pandoc import (`pandocDocxToMd`) always runs and is **not** listed in the file.

Re-run with `--force` to overwrite an existing file:

```bash
doc-convert configure --force
```

Without `--force`, the command exits with an error if `doc-convert.yml` already exists.

On success the CLI prints `Wrote …/doc-convert.yml`.

## Edit steps

Each entry under `steps` is either `run` or `skip`:

```yaml
version: 1

steps:
  improveDocxFigures: run
  improveDocxTables: skip
  improveDocxMath: skip
```

- Set `skip` to disable a step without deleting its line.
- Omit a step key — when a config file is present, omitted steps are treated as **skipped**.
- Do not add `pandocDocxToMd`; the CLI rejects it with an error.

Unknown step ids in the file produce a warning and are ignored.

### With vs without a config file

| Situation | Behavior |
|-----------|----------|
| No `doc-convert.yml` in cwd | All configurable steps are enabled (subject to `warn-skip` guards) |
| Config file present | Only steps explicitly set to `run` execute; omitted keys are skipped |
| Freshly generated file (all `run`) | Same effective pipeline as having **no** config file |

Example generated header and body (see also `docx-examples/pdc/doc-convert.yml` in the monorepo):

```yaml
# doc-convert configuration (generated by `doc-convert configure`)
#
# Pandoc import (pandocDocxToMd) always runs and is not listed here.
# Steps with `run` execute on conversion in this directory.
# Set a step to `skip` to disable it without removing the line.
# Remove a step key entirely to skip it as well.
#

version: 1
steps:
  copyDocxDependencies: run
  initMystConfig: run
  # … all configurable steps …
  enrichAffiliationsRor: run
```

## Discovery at runtime

On every conversion and `--list-steps`, the CLI reads `./doc-convert.yml` from `process.cwd()` only.

When a config is loaded, run and list output include a `Config: …/doc-convert.yml` line.

## Preview with `--list-steps`

Always lists all **13** pipeline steps with planned dispositions. Use this to verify edits before a full conversion:

```bash
cd path/to/your/project
bun ../../doc-convert/src/index.ts manuscript.docx --list-steps
```

Dispositions combine config (`run` / `skip`) with built-in guards (`warn-skip`). Details and sample output: [Pipeline](pipeline.md).
